# Composition API 的优势

##  Options API (配置式API / 选项式API)存在的问题

使用传统OptionsAPI中，新增或者修改一个需求，就需要分别在data，methods，computed里修改 。**同一个需求的数据、方法等较分散不集中。**

##  Composition API (组合式API)的优势

我们可以更加优雅的组织我们的代码，函数。让相关功能的代码更加有序的组织在一起。

> 想让组合式API发挥优势，必须借助hook函数
