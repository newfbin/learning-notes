# Dify 支持哪些向量数据库

常见支持（官方内置适配）：

- Qdrant（最常用，推荐）
- Weaviate
- Milvus
- Pinecone

👉 推荐优先级（经验）：

- 本地部署：Qdrant / Milvus
- 云服务：Pinecone
- 简单易用：Qdrant 👍

# 三、对接方式（核心步骤）

## 1️⃣ 启动向量数据库服务

### 以 Qdrant 为例（最简单）

```
docker run -p 6333:6333 qdrant/qdrant
```

启动后：

```
http://localhost:6333
```

------

## 2️⃣ 修改 Dify 配置

Dify 是通过 **环境变量**来切换向量数据库的（非常关键）

### 找到 `.env` 文件（或 docker-compose 环境变量）

修改：

```
VECTOR_STORE=qdrant
QDRANT_URL=http://localhost:6333
QDRANT_API_KEY=
```

------

### 如果你用的是其他数据库：

#### 👉 Milvus

```
VECTOR_STORE=milvus
MILVUS_HOST=localhost
MILVUS_PORT=19530
```

------

#### 👉 Weaviate

```
VECTOR_STORE=weaviate
WEAVIATE_URL=http://localhost:8080
```

------

#### 👉 Pinecone（云）

```
VECTOR_STORE=pinecone
PINECONE_API_KEY=xxx
PINECONE_ENV=xxx
```

------

## 3️⃣ 重启 Dify

```
docker-compose down
docker-compose up -d
```

------

## 4️⃣ 在 Dify 中创建知识库

进入 Dify UI：

1. 创建 Knowledge
2. 上传文档
3. 选择 embedding 模型

👉 此时：

✅ 数据会自动写入你配置的向量数据库
 ✅ 检索也走该数据库

------

# 四、验证是否成功

你可以通过以下方式确认：

### 方法1：看日志

Dify logs 中会出现：

```
Using Qdrant as vector store
```

------

### 方法2：直接查数据库

例如 Qdrant：

访问：

```
http://localhost:6333/collections
```

能看到 collection = 成功 ✅

------

# 五、一些关键细节（非常重要）

## 1️⃣ 向量维度必须匹配

比如：

- OpenAI embedding：1536维
- 本地模型：可能 768 / 1024

👉 向量数据库 collection 一旦创建：

> ❌ 维度不能改

------

## 2️⃣ Dify 会自动管理 collection

你不需要手动：

- 建表
- 建索引

👉 Dify 会自动创建：

```
dataset_xxx
```

------

## 3️⃣ chunk 策略影响很大

建议：

- 200~500 tokens / chunk
- overlap 20~50

👉 直接影响召回质量

------

## 4️⃣ 生产环境建议

- Qdrant：开启持久化
- Milvus：用集群版
- Pinecone：控制成本（按查询收费）

------

# 六、一个你可能更关心的问题

👉 **Dify 和向量数据库是“强绑定”吗？**

不是。

你可以：

- 随时切换数据库（重新导入数据即可）
- 或者直接绕过 Dify，用你自己的 RAG

------

# 七、总结一句话

👉 **Dify 对接向量数据库 = 改 `.env` + 启动数据库 + 重启服务**

它已经帮你封装好了所有复杂逻辑（embedding / schema / 查询），你只需要“接上去”。
